# 푸터 표기 + 사진 일괄 삽입 설계

## 배경

Stage 4+5까지 완료된 상태에서, 앱 최하단에 제작자 표기를 남기고 싶고, 영수증 하이라이트 사진을 매 멤버마다 개별로 첨부하는 대신 한 번에 모든 멤버에게 같은 사진을 적용하는 기능이 필요하다.

## (1) 제작자 푸터

- `<body>` 내 `#app`과 형제 위치에 정적 `<footer id="app-footer">`를 추가한다. 라우터는 `#app.innerHTML`만 교체하므로, 이 위치에 두면 모든 라우트에서 별도 처리 없이 항상 최하단에 노출된다.
- 내용: "오늘은 내가 총무" (첫 줄) / "made by JSTD" (둘째 줄, 더 작고 옅은 색).
- 스타일: 기존 컬러 팔레트(`--color-bg`, 텍스트 `#3A2E28`)와 어울리는 톤으로, 본문보다 작은 폰트, 중앙 정렬, 상하 여백 확보. 캡처 대상이 아니므로 `data-no-capture` 불필요(영수증 캡처는 `#receipt-capture` 요소만 대상으로 함).

## (2) 사진 일괄 삽입

**진입점**: `openPhotoManagerSheet` (사진 관리 시트) 안에 "전체 멤버에게 같은 사진 적용" 버튼을 추가한다 (기존 "이 모임 사진 전체 삭제" 버튼과 같은 위치 그룹).

**동작 흐름**:
1. 버튼 클릭 → 새 함수 `openBulkPhotoAttachModal(eventId)`가 `openPhotoAttachModal`과 동일한 UI 패턴(파일 선택 → `resizeImageToCanvas` → `original`/`grayscale`/`dither` 필터 프리뷰 → 확정 버튼)의 모달을 연다. 기존 `pendingPhotoCanvas`/`pendingPhotoFilter` 모듈 변수와 `applyPhotoFilter`/`canvasToCompressedDataURL`을 그대로 재사용한다.
2. 확정 버튼 클릭 시:
   - 이벤트의 멤버 중 이미 사진이 첨부된 멤버가 하나라도 있으면(`photoStore.get`으로 확인), "기존에 첨부된 사진도 모두 새 사진으로 덮어씁니다. 계속할까요?" `confirmDialog`를 띄운다. 없으면 확인창 없이 바로 진행한다.
   - 확인 후, `ev.members`를 순회하며 동일한 최종 `dataUrl`(필터 적용 + 압축 완료된 결과, 1회만 생성)로 `photoStore.set(eventId, member.id, dataUrl)`을 호출한다.
   - 도중 `QuotaExceededError` 발생 시 즉시 반복을 중단하고, `showToast`로 "N/전체 명에게 적용한 뒤 용량이 가득 찼어요"를 표시한 뒤 사진 관리 시트를 다시 연다.
   - 정상 완료 시 "전체 N명에게 사진을 적용했어요" 토스트를 표시하고 사진 관리 시트를 새로고침(재오픈)한다.
3. 모달 취소/오버레이 클릭 시 기존 개별 첨부 모달과 동일하게 아무 것도 적용하지 않고 닫는다.

**재사용 범위**: 새 코드는 "여러 멤버에게 반복 적용"하는 루프와 확인창 로직만 추가하고, 파일 읽기·리사이즈·필터·압축 로직은 기존 함수를 그대로 호출한다.

## 테스트/검증

- 수동 확인: 개발 서버 없이 정적 파일이므로 브라우저에서 직접 열어 (a) 모든 라우트 하단에 푸터가 보이는지, (b) 멤버 2명 이상인 모임에서 사진 관리 → 일괄 적용 → 필터 선택 → 확정 후 모든 멤버 영수증에 동일 사진이 반영되는지, (c) 기존에 사진이 있던 멤버가 있을 때 덮어쓰기 확인창이 뜨는지 확인한다.
- 기존 `runRegressionTests`(DEV_MODE) 스위트에는 UI 모달 로직이 포함되어 있지 않으므로 이번 변경으로 새 자동화 테스트는 추가하지 않는다(기존 photoStore CRUD 테스트는 영향 없음).
